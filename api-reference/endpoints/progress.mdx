The Progress API is the heart of UpForge's gamification loop. Every time you finish a lesson, a single `POST` request triggers XP rewards, streak tracking, and achievement evaluation â€” all in one call.

## POST /api/progress

<span style={{display:"inline-block", background:"#3b82f6", color:"#fff", borderRadius:"4px", padding:"2px 10px", fontWeight:700, fontSize:"13px", marginBottom:"12px"}}>POST</span>

Marks a lesson as completed or uncompleted. When a lesson is **newly** completed (i.e. it wasn't already marked complete), UpForge automatically:

1. **Awards XP** â€” your points increase by `floor(course.points / course.totalLessons)` for that lesson.
2. **Updates your streak** â€” daily lesson activity is recalculated and `currentStreak` / `longestStreak` are written to your profile.
3. **Evaluates achievements** â€” every unearned achievement is checked against your updated stats; any that you now qualify for are unlocked and their bonus XP is added to your total.
4. **Auto-enrolls you** â€” if you aren't already enrolled in the course that owns the lesson, an enrollment record is created for you automatically.

### Request body

<ParamField body="lessonId" type="string" required>
  The UUID of the lesson to mark complete or incomplete.
</ParamField>

<ParamField body="completed" type="boolean" required>
  Set to `true` to mark the lesson as complete, or `false` to remove the completion mark.
</ParamField>

### Example request

```json
{
  "lessonId": "lesson-uuid-here",
  "completed": true
}
```

### Response fields

<ResponseField name="success" type="boolean" required>
  `true` when the progress record was saved successfully.
</ResponseField>

### Example response

```json
{
  "success": true
}
```

<Note>
  XP is awarded only once per lesson. If you mark a lesson incomplete and then re-complete it, the system detects that the lesson was already completed previously and **does not** award XP a second time. The `isNewCompletion` flag in the route logic handles this deduplication.
</Note>

### Error responses

| Status | Meaning |
|--------|---------|
| `400`  | `lessonId` is missing from the request body. |
| `401`  | You are not authenticated. |
| `404`  | No lesson exists with the given `lessonId`, or your user record was not found. |
| `500`  | An unexpected server error occurred. |

### How XP is calculated

XP per lesson is proportional to the course's total point value:

```
XP per lesson = floor(course.points / course.totalLessons)
```

For example, a 500-point course with 20 lessons awards **25 XP** per lesson.

<Tip>
  Streak bonuses are re-evaluated on every completion. Complete at least one lesson each calendar day to keep your streak alive. The streak resets if no lesson is completed for more than one calendar day.
</Tip>

### How achievement unlocks work

After each new completion, the server compares your cumulative stats against every achievement's `criteria` object. The three criteria types currently evaluated are:

| Criteria type | Unlocks when |
|---------------|-------------|
| `lessons_completed` | Your total completed lessons â‰Ą `criteria.count` |
| `courses_completed` | Your completed enrollments â‰Ą `criteria.count` |
| `streak` | Your current streak â‰Ą `criteria.days` |

When an achievement unlocks, its `points` value is immediately added to your total XP on top of the lesson XP.
