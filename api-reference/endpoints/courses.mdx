The Courses API lets you browse every course on UpForge, inspect lesson-level detail, and manage your enrollment status. All four endpoints require a valid session — unauthenticated requests receive a `401 Unauthorized` response.

---

## GET /api/courses

Returns every course on the platform alongside your enrollment status and the total number of lessons it contains.

### Response fields

The response is an **array** of course objects.

<ResponseField name="id" type="string" required>
  UUID that uniquely identifies the course.
</ResponseField>

<ResponseField name="title" type="string" required>
  Human-readable course title.
</ResponseField>

<ResponseField name="description" type="string" required>
  Short summary of what the course covers.
</ResponseField>

<ResponseField name="thumbnail" type="string | null">
  URL for the course cover image, or `null` when no thumbnail has been set.
</ResponseField>

<ResponseField name="duration" type="number" required>
  Estimated course length in **minutes**.
</ResponseField>

<ResponseField name="points" type="number" required>
  Total XP available from completing every lesson in this course. Individual lesson XP is calculated as `floor(points / totalLessons)`.
</ResponseField>

<ResponseField name="totalLessons" type="number" required>
  Number of lessons in the course.
</ResponseField>

<ResponseField name="enrolled" type="boolean" required>
  `true` if you are currently enrolled in this course, `false` otherwise.
</ResponseField>

<ResponseField name="progress" type="number" required>
  Always `0` in this response. For your real per-course completion percentage, call [GET /api/user/stats](/api-reference/endpoints/user#get-apiuserstats) and inspect `coursesInProgress` / `completedCourses`.
</ResponseField>

<Note>
  The `progress` field in this response is a placeholder. Use `GET /api/user/stats` to retrieve accurate progress numbers.
</Note>

### Error responses

| Status | Meaning |
|--------|---------|
| `401`  | You are not authenticated. |

### Example response

```json
[
  {
    "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "title": "Full-Stack JavaScript",
    "description": "Build real apps with React and Node.js",
    "thumbnail": "https://img.youtube.com/vi/xyz/maxresdefault.jpg",
    "duration": 240,
    "points": 500,
    "totalLessons": 20,
    "enrolled": true,
    "progress": 0
  },
  {
    "id": "b2c3d4e5-f6a7-8901-bcde-f12345678901",
    "title": "Python for Data Science",
    "description": "Analyze data with pandas, NumPy, and Matplotlib",
    "thumbnail": null,
    "duration": 180,
    "points": 400,
    "totalLessons": 15,
    "enrolled": false,
    "progress": 0
  }
]
```

## GET /api/courses/:courseId

Returns the full details for a single course, including its ordered lesson list and your per-lesson completion status.

### Path parameters

<ParamField path="courseId" type="string" required>
  The UUID of the course you want to retrieve.
</ParamField>

### Response fields

<ResponseField name="id" type="string" required>
  UUID that uniquely identifies the course.
</ResponseField>

<ResponseField name="title" type="string" required>
  Human-readable course title.
</ResponseField>

<ResponseField name="description" type="string" required>
  Short summary of what the course covers.
</ResponseField>

<ResponseField name="thumbnail" type="string | null">
  URL for the course cover image, or `null` when no thumbnail has been set.
</ResponseField>

<ResponseField name="duration" type="number" required>
  Estimated course length in **minutes**.
</ResponseField>

<ResponseField name="difficulty" type="string" required>
  Difficulty label for the course (e.g. `"beginner"`, `"intermediate"`, `"advanced"`).
</ResponseField>

<ResponseField name="points" type="number" required>
  Total XP available from completing every lesson in this course.
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  ISO 8601 timestamp of when the course was created.
</ResponseField>

<ResponseField name="updatedAt" type="string" required>
  ISO 8601 timestamp of the last update to the course.
</ResponseField>

<ResponseField name="enrolled" type="boolean" required>
  `true` if you are enrolled in this course.
</ResponseField>

<ResponseField name="completed" type="boolean" required>
  `true` if your enrollment record is marked complete (i.e. you finished the course).
</ResponseField>

<ResponseField name="lessons" type="Lesson[]" required>
  Lessons sorted in ascending display order.

  <Expandable title="Lesson object fields">
    <ResponseField name="id" type="string">UUID of the lesson.</ResponseField>
    <ResponseField name="courseId" type="string">UUID of the parent course.</ResponseField>
    <ResponseField name="title" type="string">Lesson title.</ResponseField>
    <ResponseField name="description" type="string | null">Optional lesson description.</ResponseField>
    <ResponseField name="content" type="string">Main lesson body text or HTML.</ResponseField>
    <ResponseField name="videoUrl" type="string | null">Embedded video URL, or `null` if the lesson is text-only.</ResponseField>
    <ResponseField name="order" type="number">1-based position of the lesson within the course.</ResponseField>
    <ResponseField name="completed" type="boolean">Whether you have marked this lesson complete. Always `false` for unenrolled users.</ResponseField>
    <ResponseField name="createdAt" type="string">ISO 8601 timestamp of when the lesson was created.</ResponseField>
    <ResponseField name="updatedAt" type="string">ISO 8601 timestamp of the last update.</ResponseField>
  </Expandable>
</ResponseField>

### Error responses

| Status | Meaning |
|--------|---------|
| `401`  | You are not authenticated. |
| `404`  | No course exists with the given `courseId`, or your user record was not found. |
| `500`  | An unexpected server error occurred. |

## POST /api/courses/:courseId/enroll

Enrolls you in the specified course. No request body is required.

### Path parameters

<ParamField path="courseId" type="string" required>
  The UUID of the course to enroll in.
</ParamField>

### Response fields

<ResponseField name="success" type="boolean">
  `true` when a new enrollment was created. Omitted when you were already enrolled.
</ResponseField>

<ResponseField name="enrolled" type="boolean">
  Returned only when you were already enrolled. Always `true` in that case, alongside a `message` field.
</ResponseField>

<ResponseField name="enrollment" type="object">
  The newly created enrollment record. Only present when a new enrollment was made.

  <Expandable title="Enrollment object fields">
    <ResponseField name="id" type="string">UUID of the enrollment record.</ResponseField>
    <ResponseField name="userId" type="string">Your internal UpForge user ID.</ResponseField>
    <ResponseField name="courseId" type="string">The enrolled course's UUID.</ResponseField>
    <ResponseField name="enrolledAt" type="string">ISO 8601 timestamp of enrollment.</ResponseField>
    <ResponseField name="completed" type="boolean">Always `false` on a new enrollment.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable status message.
</ResponseField>

### Error responses

| Status | Meaning |
|--------|---------|
| `401`  | You are not authenticated. |
| `404`  | No course exists with the given `courseId`, or your user record was not found. |
| `500`  | An unexpected server error occurred. |

### Example — new enrollment

```json
{
  "success": true,
  "enrollment": {
    "id": "enroll-uuid-here",
    "userId": "user-uuid-here",
    "courseId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "enrolledAt": "2024-11-01T12:00:00.000Z",
    "completed": false
  },
  "message": "Successfully enrolled in course"
}
```

### Example — already enrolled

```json
{
  "message": "Already enrolled",
  "enrolled": true
}
```

## DELETE /api/courses/:courseId/enroll

Removes your enrollment from a course. Any lesson progress you have recorded is **not** deleted.

### Path parameters

<ParamField path="courseId" type="string" required>
  The UUID of the course to unenroll from.
</ParamField>

### Response fields

<ResponseField name="success" type="boolean">
  `true` when the enrollment record was removed.
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable confirmation message.
</ResponseField>

### Error responses

| Status | Meaning |
|--------|---------|
| `401`  | You are not authenticated. |
| `404`  | Your user record was not found. |
| `500`  | An unexpected server error occurred. |

### Example response

```json
{
  "success": true,
  "message": "Successfully unenrolled from course"
}
```

<Warning>
  Unenrolling does not delete your lesson progress records. If you re-enroll later, your previously completed lessons will still appear as completed.
</Warning>
