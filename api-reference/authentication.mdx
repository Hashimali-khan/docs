The UpForge API uses session-based authentication. Every endpoint verifies your signed-in session before returning data — unauthenticated requests receive a `401 Unauthorized` response. UpForge uses [Clerk](https://clerk.com) under the hood to manage identity and sessions.

## Browser Requests

When you're signed into the UpForge web app, your browser automatically stores and sends the session cookie with every API call. You don't need to set any extra headers — just make sure you pass `credentials: 'include'` in your `fetch` call so the cookie is forwarded.

```javascript browser-fetch.js
const res = await fetch('/api/courses', {
  credentials: 'include', // include session cookies
});
const courses = await res.json();
```

<Tip>
  When building browser-based integrations, always use `credentials: 'include'` in your fetch calls to automatically send the auth cookie.
</Tip>

## Server-side / Programmatic Access

For server-side scripts or integrations running outside the browser, your session cookie is not available automatically. Instead, you need to pass a valid session token in the `Authorization` header. UpForge uses Clerk for authentication — obtain a session token from the [Clerk dashboard](https://dashboard.clerk.com) or via the Clerk SDK in your backend.

```javascript server-fetch.js
const res = await fetch('https://your-upforge-instance.com/api/courses', {
  headers: {
    'Authorization': 'Bearer YOUR_SESSION_TOKEN',
    'Content-Type': 'application/json',
  },
});
const courses = await res.json();
```

Replace `YOUR_SESSION_TOKEN` with a short-lived session token issued by Clerk for your account.

<Warning>
  Never expose session tokens in client-side code or commit them to source control. Treat them with the same care as passwords.
</Warning>

## Handling 401 Errors

If any API call returns a `401` response, your session has expired or is no longer valid. Sign in again to get a fresh session, then retry the request.

```json
"Unauthorized"
```

This plain-text response body is returned for every unauthenticated request across all `/api/*` routes.

## Protected Routes

UpForge enforces authentication at the middleware level using Clerk. The following are always protected:

- **`/api/*`** — all API routes require a valid session. Unauthenticated requests receive a `401` response immediately.
- **`/dashboard/*`** — all dashboard pages require a valid session. Unauthenticated requests are automatically redirected to the sign-in page.

The only publicly accessible routes are `/`, `/sign-in`, `/sign-up`, and `/api/public/*`. Everything else requires you to be signed in before you can access it.
